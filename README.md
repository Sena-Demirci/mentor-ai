# Mentor AI
###
> Think. Build. Learn.

>  **Status:** MVP (CLI Version) – Active Development
### **Project Overview**
The purpose of this project is not to generate ready-made code, but to help users learn programming by encouraging them to think, solve problems, and write the code themselves with AI-guided mentoring.
###
### The Core Philosophy of the Project

- **The Trap of the "Easy Way"**: People naturally prefer the easiest solution, and AI-generated code can be very tempting. However, relying on ready-made solutions too often weakens the problem-solving and learning process.

- **Avoiding AI is Not the Answer**: The goal is not to stop using artificial intelligence. AI is a powerful tool and should be embraced, not avoided.

- **AI as an Assistant, Not a Replacement**: We must use AI to our advantage. AI should support our thinking, not replace it. Instead of solving every problem for us, it should guide us, ask questions, and help us reach the solution ourselves.

- **The Mission**: Mentor AI is built around this philosophy: encouraging active learning by turning AI into a programming mentor rather than a code generator.

### **Current Features**
- AI-powered programming mentor.

- Modular AI client architecture for future model integrations.

- Hint Mode that provides guidance instead of direct solutions.

- Guiding Question Mode to encourage critical thinking.

- Solution Mode for users who need a complete explanation.

- General AI chat for programming-related questions.

- Command Line Interface (CLI) prototype.
